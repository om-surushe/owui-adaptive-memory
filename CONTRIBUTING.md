# Contributing to Adaptive Memory

Thank you for your interest in contributing to Adaptive Memory! This plugin helps OpenWebUI users have more personalized AI conversations through persistent memory.

## How to Contribute

### Reporting Issues

- Check existing issues before creating a new one
- Include your OpenWebUI version and Python version
- Provide clear steps to reproduce the issue
- Share relevant log output (sanitize any sensitive data)

### Submitting Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes to `adaptive_memory_v3.1.py`
4. Test your changes locally in OpenWebUI
5. Commit with clear messages: `git commit -m "feat: add new feature"`
6. Push and open a Pull Request

### Code Style

- Follow PEP 8 guidelines
- Use type hints where applicable
- Add docstrings for new functions/methods
- Keep functions focused and modular

## Development Setup

1. Clone your fork
2. Install in your OpenWebUI instance as a Filter plugin
3. Enable debug logging in Valves for development
4. Test with various LLM and embedding providers

## Questions?

Open an issue with the `question` label if you need help!
